# 业务信息收集管理系统

一个基于微信小程序和云开发的企业级业务信息收集和管理系统，支持多角色权限管理、智能酬金计算和数据导出功能。

## 🚀 项目特性

### 核心功能
- **多角色权限管理** - 支持销售人员、片区经理、销售部门三种角色
- **智能业务录入** - 支持手动录入和快速解析两种方式
- **酬金自动计算** - 基于FTTR业务模型的智能酬金计算
- **图片管理** - 支持多图片上传和管理
- **数据筛选导出** - 强大的数据筛选和批量导出功能
- **实时统计** - 多维度数据统计和可视化展示

### 技术特色
- **云原生架构** - 基于微信云开发，无需服务器运维
- **响应式设计** - 适配各种屏幕尺寸，优秀的用户体验
- **模块化开发** - 高度模块化的代码结构，易于维护和扩展
- **安全可靠** - 严格的权限控制和数据安全保护

## 🏗️ 快速开始

### 环境要求
- 微信开发者工具 最新版本
- 微信小程序账号
- 微信云开发环境

### 安装步骤

1. **克隆项目**
   ```bash
   git clone [项目地址]
   cd miniprogram-2
   ```

2. **配置小程序**
   - 在微信开发者工具中导入项目
   - 修改 `project.config.json` 中的 `appid`
   - 配置云环境ID：`cloud1-1g65i23w93c28d35`

3. **部署云函数**
   - 右键点击各云函数目录选择"创建并部署：云端安装依赖"
   - 依次部署：`userManagement`、`businessData`、`fileUpload`

4. **初始化数据库**
   - 在云开发控制台创建数据库集合：`users`、`business_records`
   - 执行 `cloudfunctions/database-init.js` 初始化索引
   - （可选）执行 `cloudfunctions/test-data-generator.js` 生成测试数据

5. **配置权限**
   - 在云开发控制台设置数据库权限为"仅创建者可读写"
   - 配置云存储权限

## 👥 用户角色说明

### 销售人员 (sales_person)
- **权限**：创建和查看自己的业务记录
- **功能**：信息录入、图片上传、查看个人统计

### 片区经理 (district_manager)  
- **权限**：查看和管理下属销售人员的记录
- **功能**：业务审核、数据筛选、区域统计

### 销售部门 (sales_department)
- **权限**：查看和管理所有业务记录
- **功能**：全局管理、数据导出、系统配置

## 💼 业务流程

### 1. 信息收集流程
```
用户登录 → 选择收集页面 → 填写业务信息 → 上传相关图片 → 系统计算酬金 → 提交审核
```

### 2. 审核管理流程
```
管理员登录 → 查看待审核记录 → 筛选和查看详情 → 审核通过/拒绝 → 添加审核意见
```

### 3. 数据导出流程
```
管理员权限验证 → 设置筛选条件 → 选择导出格式 → 生成导出文件 → 下载数据
```
